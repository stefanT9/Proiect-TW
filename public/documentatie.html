Abstract

Numele proiectului PiRo vine de la Pisa Romania si reprezinta o platforma web pe care se pot vizualiza diferite statistici referitoare la rezultatele Romaniei la testele Pisa (Programme for International Student Assessment)

Introducere

Testele Pisa masoara capacitatea unui copil de 15 ani de a citi, de a face calcule si de a vedea cunostintele despre lumea inconjuratoare. Mai multe despre aceste teste puteti afla aici.

Motivatie

O astfel de platforma va putea ajuta viitoarele generatii intr-o pregatire mai buna pentru aceste teste, astfel vizualizarea rezultatelor testelor trecute ii vor putea ajuta in a determina care sunt ariile cu rezultate mai putin bune pentru participantii din Romania.

Structura

Platforma noastra a fost impartita in trei pagini HTML, fiecare avand propriul scop si propriile functionalitati.

Pagina home

Aceasta pagina are rol de informare a utilizatorilor veniti pe platforma cu privire la scopul platformei.

Arhitectura

Abordare bazata pe microservicii

Arhitectura aleasa pentru aceasta platforma a fost cea de spargere in microservicii, astfel functionalitatile total independente apartin unor microservicii diferite.

Users

Acest microserviciu se va ocupa strict de autentificarea userilor pe platforma noastra.
Totodata, se va ocupa si de gestionarea acestora, apeland la un CRUD pentru baza de date.

Columns

Acest microserviciu se va ocupa de managementul filtrelor/coloanelor ce pot aparea pe platforma.
Acestea pot fi modificate doar daca autentificarea userului s-a realizat cu succes inainte.

Values

Acest microserviciu se va ocupa de managementul valorilor ce apar in datele noastre pentru fiecare coloana.
Acestea pot fi modificate doar daca autentificarea userului s-a realizat cu succes inainte.

Resources

Acest microserviciu se va ocupa de managementul resurselor necesare requesturilor clientilor (resurse HTML, CSS, JS, imagini), orice nu este legat de API.

Fat client

Pentru o experienta mai buna pentru utilizator si pentru evitarea suprasolicitarii serverului, aceasta platforma va incarca dinamic toate elementele web ale requestului.

Implementare

Front-end

Stack folosit: HTML5, CSS3, Javascript (Vanilla)
Arhitectura fiind bazata pe fat-client, se trimite doar un template de pagina care ulterior este incarcat dinamic.
Un exemplu de astfel de incarcare dinamica este cea a filtrelor, in care pe baza tipului de filtru (continuu sau discret), se va incarca un sablon templatizat.

for each column in columns
  if column.type is "discrete" then appendDiscreteFilter(column)
  else appendContinousFilter(column)

Back-end

Stack folosit: NodeJs
Pentru o eficienta sporita, marea majoritate a functiilor sunt asincrone.
Totodata, acest stack a fost ales datorita dimensiunilor de input si output, iar urma unui research facut de fiecare, am hotarat ca NodeJs este alegerea cea mai buna pentru aceasta platforma.

Baza de date

Datele platformei sunt salvate sub forma de colectii in MongoDB deoarece acest tip de baze de date nerelationale fac cautarea in spatiul de date mult mai rapida.
Momentan avem trei colectii, dupa cum urmeaza:

  • Colectia User

    • In aceasta colectie sunt stocati toti utilizatorii platformei. Fiecare user are un email si o parola criptata.

  • Colectia Columns

    • In aceasta colectie sunt stocate date despre fiecare coloana. Fiecare coloana are informatii precum: numele acesteia, tipul (discreta sau continua) etc.

  • Colectia Values

    • In aceasta colectie sunt stocate date despre valorile fiecarei coloane. Astfel, fiecare coloana va avea cate un array de valori (de exemplu: coloana CNTRYID va contine ID-urile unor tari)
